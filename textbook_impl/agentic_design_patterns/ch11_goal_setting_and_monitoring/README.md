# Chapter 11: Goal Setting and Monitoring Pattern (目标设定与监控模式)

## 📖 概述

本章节展示了**目标设定与监控模式**的实现，这是 Agentic Design Patterns 中的重要模式之一。该模式让智能体能够：

- 设定明确的目标
- 将大目标分解为可衡量的子目标
- 持续监控进展
- 根据反馈进行动态调整
- 确保最终目标的达成

## 🎯 核心功能

### 1. 目标分解 (Goal Decomposition)
使用 LLM 将复杂的大目标智能分解为 3-5 个具体的、可衡量的子目标，每个子目标包含：
- 清晰的描述
- 合理的截止日期
- 2-3 个关键衡量指标

### 2. 进度跟踪 (Progress Tracking)
- 实时更新子目标的完成进度（0-100%）
- 自动计算总体进度
- 记录反馈意见和备注信息
- 根据进度自动更新状态

### 3. 状态监控 (Status Monitoring)
- 生成详细的进度报告
- 统计各状态的子目标分布
- 识别需要关注的子目标
- 提供可视化的进度概览

### 4. 动态调整 (Dynamic Adjustment)
- 分析当前执行情况
- 提供智能调整建议：
  - 优先级调整
  - 资源重新分配
  - 时间延长
  - 策略改变
- 应用调整并更新计划

### 5. 数据持久化 (Data Persistence)
- 支持保存目标计划到 JSON 文件
- 支持从文件加载已保存的计划
- 完整的状态序列化/反序列化

## 📋 示例场景

本示例演示了一个"30天掌握 Python 编程基础"的学习规划场景：

1. **创建目标计划**：将学习目标分解为 5 个子目标
   - 基本语法和数据结构
   - 函数和模块
   - 面向对象编程
   - 文件操作和异常处理
   - 综合项目实践

2. **模拟进度更新**：更新部分子目标的进度

3. **生成进度报告**：查看当前的整体执行情况

4. **获取调整建议**：LLM 分析并提供 5 条优化建议

5. **应用调整**：将建议应用到目标计划中

6. **保存计划**：将完整的目标计划保存到 JSON 文件

## 💡 关键技术点

### Prompt 模板设计
- 使用 `ChatPromptTemplate` 构建结构化提示
- 在 human message 中包含 JSON 格式示例
- 使用双花括号 `{{}}` 转义 JSON 中的花括号

### 数据模型
- 使用 `dataclass` 定义清晰的数据结构
- 使用 `Enum` 管理目标状态
- 支持嵌套的子目标列表

### LLM 集成
- 使用 `JsonOutputParser` 解析结构化输出
- 通过 chain 组合 prompt、LLM 和 parser
- 支持中文内容的 JSON 序列化

## 📊 输出示例

程序会输出：
- ✅ 分解后的子目标列表
- 📈 实时进度报告
- 💡 智能调整建议
- 💾 保存的 JSON 文件
